# Instrucciones permanentes para Codex

- Trabajar siempre en espanol claro y practico.
- Mantener la estructura semanal del curso.
- Priorizar practica, ejemplos ejecutables, ejercicios y proyectos.
- Crear ejercicios autocontenidos en el archivo `.py` con docstring inicial, instrucciones, pistas breves y `ZONA DE TRABAJO`.
- Mantener archivos `.md` como respaldo o explicacion extendida, no como requisito para resolver.
- Separar soluciones y explicaciones en `soluciones/`.
- No poner soluciones dentro de los enunciados.
- No poner soluciones dentro de los archivos `.py` de ejercicios.
- Crear ejercicios graduales: basicos, intermedios, aplicados y retos.
- Usar pytest en ejercicios de Python puro cuando sea razonable.
- Para hardware, incluir version simulada sin hardware y advertencias.
- Priorizar ESP32 con MicroPython.
- Usar Flet para dashboard de PC.
- Usar web local servida por ESP32 para celular.
- Documentar supuestos en `SUPUESTOS.md`.
- Cuidar seguridad electrica y logica de 3.3 V.
